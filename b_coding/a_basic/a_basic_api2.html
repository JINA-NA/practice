div는 의미가 없는 단순한 영역을 나타나게 하는 박스의 의미를 가진다.

span요소는 의미가 없는 단순한 내용의 그룹. div와는 조금 다른 의미다.

시멘틱 구조는 각 영역별로 그 의미를 가지고 쓰이도록 처리된 영역

hedear, nav, section, article, aside, main, footer, figure

영역을 나타내는 의미를 구조

시멘틱 구조에 대해서 알아보자

header : 웹페이지의 상단에 존재, 웹의 로고,h1을 담으면서, 전체 웹페이지 구조를 파악하는 nav영역을 함께 포함시키는 기능을 가진다.

navigation

nav : 웹페이지 내부에서 클릭하면 해당하는 페이지로 이동하게 만들어주는 링크들의 집합. navigation 요소를 담아놓은 영역

하나의 대표 내용을 보여주는 영역

section : 하나의 대표 영역을 보여주는 영역으로, 다른 내용과 연계가 되어있다. 이 뜻이 뭐냐, 배너를 클릭하면 그 부분을 상세페이지로 넘어가는 걸 뜻한다. 본문. ex) 광고 배너

별도의 독립영역으로 설정되어도 문제 없는 영역

article : section과 유사한 기능을 가진 영역. 본문 내용 또는 다른 내용과 연계가 되지 않아도 별도의 독립영역으로 설정되어도 문제 없는 영역. 이 부분이 section 태그와 다른점. ex)게시판 안내문, 신문기사, 공고

aside : another 영역을 지징하는 것. 사이트의 구성요소에 없어도 전혀 문제 되지 않는 영역. 그래도 있으면 도움이 되는 영역. ex)쇼핑몰에 옆에 보면 최근 본 상품 영역 같은 거. 없어도 문제는 없지만 있으면 편리함.

footer 입니다.

footer : 이름으로만 보면 하단에 존재한다는 의미를 가지고 있지만, 실제로는 하단에만 존재하는 건 아니다.
footer는 회사 혹은 사이트의 정보를 담고 있는 영역을 말한다. ex)주소, 전화번호, 사이트 제작연도, 지침, 소비자 관련 정보 등, 패밀리 사이트
정보가 보이지 않는다고해도 회사 또는 사이트에 대한 내용을 반드시 존재해야한다.

main : section과 article의 영역을 정리해서 더 큰 영역으로 담는 곳. 그런데 아직 대부분의 사이트에서는 완벽하게 지원하지 못하고 있는 것이 현실. 익스플로러는 거의 안 되고 있는 실정.
